        Schema::table('posts', function (Blueprint $table) {

        });

     * Reverse the migrations.

            $table->dropIndex('posts_status_published_index');

            $table->dropIndex('taggables_reverse_lookup');

Key optimization targets:
- `scopePublished()` uses `where('status', 'published')` - needs status index
- Homepage query: `Post::published()->latest('published_at')` - needs compound index
- Tag/category pages load posts with polymorphic relations - needs reverse lookup indexes
